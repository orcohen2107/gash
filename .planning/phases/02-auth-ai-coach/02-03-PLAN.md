**JWT Validation (from 02-RESEARCH.md Pattern 2):**

- Create Supabase client with token: `createClient(URL, ANON_KEY, { global: { headers: { Authorization: 'Bearer {token}' } } })`
- Call `supabase.auth.getUser()` to validate token
- If validation fails (error or !user): return 401 Unauthorized
- On success: user.id is authorized, use for RLS

Create client-side wrapper around Supabase Edge Function invocation:

```typescript
import { supabase } from './supabase'
import { ChatMessage } from '@/types'

export async function callCoach(messages: ChatMessage[]) {
  try {
    const { data, error } = await supabase.functions.invoke('ask-coach', {
      body: { messages },
    })

    if (error) throw error
    if (!data?.content) throw new Error('No response from coach')

    return {
      success: true,
      content: data.content,
    }
  } catch (err) {
    return {
      success: false,
      error: err instanceof Error ? err.message : 'Unknown error',
    }
  }
}
```

- Export: callCoach(messages: ChatMessage[]): Promise<{ success: boolean, content?: string, error?: string }>
- JWT is auto-forwarded by supabase.functions.invoke (no manual header needed)
- Error handling: catch network errors, API errors, timeout
- Caller (useChatStore, later plans) receives { success, content, error }
