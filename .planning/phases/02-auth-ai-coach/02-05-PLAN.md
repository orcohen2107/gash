   - User types and taps send in coach.tsx
   - Call useChatStore.sendMessage(text)
   - Action:
     a) Insert user message to chat_messages: role='user', content=text
     b) Store in useChatStore.messages immediately (optimistic UI not needed for MVP, but can add)
     c) Call callCoach(messages) from lib/claude.ts to invoke Edge Function

   - Function returns { content: response }
   - useChatStore.sendMessage receives response
   - Store updates messages array (re-fetch or add to array)
   - Coach.tsx FlatList re-renders

export const useChatStore = create<ChatStoreState>((set) => ({
  // ... existing state ...

  loadHistory: async () => {
    set({ loading: true })
    try {
      const { data, error } = await supabase
        .from('chat_messages')
        .select('id, role, content, created_at')
        .order('created_at', { ascending: true })

      if (error) throw error

      set({
        messages: data.map(msg => ({
          id: msg.id,
          role: msg.role as 'user' | 'assistant',
          content: msg.content,
          createdAt: msg.created_at,
        })),
        loading: false,
      })
    } catch (err) {
      console.error('Failed to load chat history:', err)
      set({ loading: false })
      Toast.show({
        type: 'error',
        text1: 'בעיה בטעינה',
        text2: 'לא הצלחנו לטעון את ההסטוריה',
      })
    }
  },
}))
```

- Action: async function that queries chat_messages, orders by created_at ascending (oldest first)
- Set loading: true before query, false after
- Transform DB row to ChatMessage type (id, role, content, createdAt)
- On error: log, show toast, clear loading
- RLS automatically filters to current user (no manual WHERE user_id needed)

export const useChatStore = create<ChatStoreState>((set, get) => ({
  // ... existing actions ...

  sendMessage: async (text: string) => {
    if (!text.trim()) return // Validate non-empty

    set({ loading: true })

    try {
      // 1. Insert user message to DB
      const { error: insertUserError } = await supabase
        .from('chat_messages')
        .insert({
          role: 'user',
          content: text.trim(),
          created_at: new Date().toISOString(),
        })

      if (insertUserError) throw insertUserError

      // 2. Add user message to UI immediately (optimistic)
      const userMessage: ChatMessage = {
        id: `temp-user-${Date.now()}`, // Temp ID until server assigns
        role: 'user',
        content: text.trim(),
        createdAt: new Date().toISOString(),
      }
      set(state => ({ messages: [...state.messages, userMessage] }))

      // 3. Call Edge Function to get AI response
      const currentMessages = get().messages
      const response = await callCoach(currentMessages)

      if (!response.success) {
        throw new Error(response.error || 'Failed to get response')
      }

      // 4. Save AI response to DB (Edge Function already does this, but add for idempotence)
      // Actually: Edge Function saves AI response, so just add to local state
      const aiMessage: ChatMessage = {
        id: `temp-assistant-${Date.now()}`, // Will be updated on re-fetch
        role: 'assistant',
        content: response.content!,
        createdAt: new Date().toISOString(),
      }
      set(state => ({ messages: [...state.messages, aiMessage] }))

      set({ loading: false })
    } catch (err) {
      console.error('Failed to send message:', err)
      set({ loading: false })
      Toast.show({
        type: 'error',
        text1: 'בעיה בשליחה',
        text2: 'לא הצלחנו לשלוח את ההודעה',
      })
    }
  },
}))
```

Flow:
1. Validate non-empty message
2. Insert user message to chat_messages table (role='user')
3. Add to local messages array (optimistic UI)
4. Call callCoach (Edge Function) with current messages
5. Edge Function returns AI response (already saved to DB by Edge Function)
6. Add AI message to local messages array
7. Coach.tsx FlatList re-renders automatically

Error handling:
- If insert fails: show toast "בעיה בשליחה"
- If Edge Function fails: show toast "בעיה בשליחה"
- Set loading: false in finally

Per CONTEXT.md: "No optimistic UI required for MVP, but local update doesn't hurt"
Per CLAUDE.md: "Edge Function saves AI response to chat_messages, client doesn't need to"
